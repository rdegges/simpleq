# Security Policy

## Supported versions

Security fixes are applied to the latest released `2.x` line.

## Reporting a vulnerability

Do not open a public GitHub issue for a suspected security issue.

Instead:

1. Email the maintainer directly using the address published in the project metadata.
2. Include reproduction details, impact, and any suggested mitigation.
3. Allow time for triage and coordinated disclosure before publishing details.

## Scope

Report issues involving:

- credential leakage
- unsafe queue or IAM defaults
- message spoofing or validation bypasses
- dashboard or CLI vulnerabilities
